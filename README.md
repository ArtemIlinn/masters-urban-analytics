## Проект 1 — Анализ застройки и землепользования

**Цель.** Разобрать структуру землепользования Хамовников и посчитать, какая доля
территории района занята застройкой.

**Ход работы (9 шагов):**

1. Загрузка зданий и зон землепользования из OSM
2. Разведочный анализ данных
3. Обработка: оставляю только полигоны и нужные поля
4. Перепроецирование в метрическую систему (UTM, для Москвы — `EPSG:32637`)
5. Расчёт доли застроенной территории (по объединению геометрий, чтобы не задваивать
   наложения зданий)
6. Разбивка землепользования по типам и расчёт долей
7. Карты зданий и землепользования + круговая диаграмма структуры
8. Сохранение очищенных слоёв в GeoPackage
9. Выводы

**Что получилось:**

| Показатель                                 | Значение                                             |
| ---------------------------------------------------- | ------------------------------------------------------------ |
| Площадь района                          | ~10 км²                                                   |
| Зданий учтено                            | ~2 800                                                       |
| Доля застроенной территории | **~20%**                                               |
| Размечено зонами `landuse`          | ~61% площади района                             |
| Преобладающий тип                    | `residential` (~64% размеченной площади) |

**Вывод.** Для центрального района застройка умеренная (~20%): между домами много
дворов, скверов и широких улиц. В размеченной части резко преобладает жилая функция,
дальше идут `commercial` и зелёные зоны. То есть Хамовники — прежде всего жилой район с
заметным деловым и зелёным каркасом.

## Проект 2 — Оценка пространственной доступности

**Цель.** Оценить пешеходную доступность двух категорий объектов — **школ** и **детских
садов** — для жителей района: построить зоны доступности, прикинуть население жилых домов
по жилой площади и посчитать, какая доля жителей оказывается внутри и вне зон.

**Ход работы (9 шагов):**

1. Загрузка границы района, объектов двух категорий и жилых зданий
2. Разведочный анализ данных
3. Подготовка слоёв: объекты → точки, здания → жилые полигоны
4. Перепроецирование в метрическую систему (UTM)
5. Оценка населения по жилой площади
6. Построение зон пешеходной доступности (буфер с нормативным радиусом)
7. Расчёт населения внутри и вне зон
8. Карты доступности по каждой категории
9. Выводы

**Как считаю население.** По жилой площади из OSM:

```
площадь застройки = площадь полигона здания
площадь этажей    = площадь застройки × этажность
жилая площадь     = площадь этажей × 0.70   (доля полезной площади)
население         = жилая площадь / 30 м²   (норма на человека)
```

Этажность беру из тега `building:levels`, где его нет — подставляю значение по умолчанию.

**Радиусы доступности** (по градостроительным нормам): школа — **500 м**, детский сад —
**300 м**.

**Что получилось:**

| Категория      | Радиус | Жителей в зоне | Жителей вне зоны                       |
| ----------------------- | ------------ | -------------------------- | ---------------------------------------------------- |
| Школы              | 500 м       | ~99%                       | **~1%** (~1,3 тыс. чел., 3 дома)     |
| Детские сады | 300 м       | ~72%                       | **~28%** (~34 тыс. чел., 209 домов) |

*Оценка населения района — около 119 тыс. человек (641 жилой дом, средняя этажность ~7).*

**Вывод.** По школам район обеспечен почти полностью. По детским садам ситуация заметно
хуже — вне пешеходной доступности оказывается около четверти жителей. Дело прежде всего в
радиусе: 300 м покрывают гораздо меньшую площадь, чем 500 м, даже при том что садов в
данных больше, чем школ. На картах дефицит виден по краям района — именно туда логично
добавлять новые детские сады.

## Данные и ограничения

- **Источник данных** — OpenStreetMap (через `osmnx`). Полнота и аккуратность разметки
  влияют на все цифры, поэтому результаты стоит читать как оценку, а не как точный
  кадастр.
- **Население** в проекте 2 оценено по жилой площади с допущениями об этажности и норме
  жилой площади на человека — это приближение.
- **Буфер по прямой** немного завышает доступность по сравнению с реальными пешеходными
  маршрутами (реки, железные дороги, заборы). Точнее было бы считать сетевые изохроны по
  графу дорог и брать жилую площадь из фактических данных.
